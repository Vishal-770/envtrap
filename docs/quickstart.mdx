## Prerequisites

- Node.js **18+**
- npm or pnpm

## Step 1 — Install

Install the `envtrap` CLI globally:

```bash
npm install -g envtrap
```

Or use it without installing via npx:

```bash
npx envtrap run node app.js
```

## Step 2 — Run your app through envtrap

Simply prefix your existing start command with `envtrap run`:

```bash
# Before
node app.js

# After — protected
envtrap run node app.js
```

That's it. envtrap now:
- Intercepts all outbound HTTPS requests
- Scans stdout / stderr for secrets
- Monitors child processes and DNS queries

## Step 3 — Watch the terminal output

When envtrap detects a secret or suspicious channel, it prints a real-time alert directly to your terminal:

```
envtrap terminal
$ envtrap run node app.js

  BLOCKED
  Channel:      HTTPS
  Secret:       STRIPE_SECRET_KEY
  Fingerprint:  SHA256:a4b4ecc7d4a4...
  Destination:  api.attacker.com
  Action:       Request terminated
```

## Step 4 — (Optional) Create a config file

For fine-grained control, create an `envtrap.json` in your project root:

```json
{
  "channels": {
    "stdout": "warn",
    "stderr": "warn",
    "network": "block",
    "child_process": "warn",
    "dns": "block"
  },
  "exclusions": {
    "domains": ["api.stripe.com", "api.openai.com"],
    "paths": ["test/**"]
  },
  "entropy": {
    "threshold": 3.5,
    "minLength": 12
  }
}
```

<Tip>
  Use `"warn"` mode for channels during development to log detections without blocking, then switch to `"block"` in production.
</Tip>

## Step 5 — Integrate with package.json

Add envtrap to your npm start script so every run is protected:

```json
{
  "scripts": {
    "start": "envtrap run node app.js",
    "dev": "envtrap run node --watch app.js"
  }
}
```

## What Happens on Detection?

| Channel | Action (in `block` mode) | Action (in `warn` mode) |
|---|---|---|
| `network` | Request terminated (via local MITM proxy) | Warning logged, request allowed |
| `stdout` / `stderr` | Process terminated immediately | Output redacted, warning logged |
| `child_process` | Subprocess spawn blocked (throws error) | Warning logged, subprocess allowed |
| `dns` | DNS query dropped (throws error) | Warning logged (or entropy warning shown) |
