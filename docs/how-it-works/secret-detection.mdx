## Detection Methods

envtrap uses two complementary techniques to detect secrets in flight:

1. **Pattern Matching** — regex patterns for known secret formats
2. **Shannon Entropy Analysis** — statistical detection of high-entropy strings

## Pattern Matching

envtrap ships with a built-in library of patterns for the most common secret types:

| Provider | Pattern | Example Match |
|---|---|---|
| Stripe | `sk_live_[0-9a-zA-Z]{24,}` | `sk_live_abc123...` |
| AWS | `AKIA[0-9A-Z]{16}` | `AKIAIOSFODNN7EXAMPLE` |
| GitHub | `ghp_[a-zA-Z0-9]{36}` | `ghp_abc123...` |
| OpenAI | `sk-[a-zA-Z0-9]{48}` | `sk-proj-...` |
| Google | `AIza[0-9A-Za-z-_]{35}` | `AIzaSy...` |
| Slack | `xox[baprs]-[0-9a-zA-Z-]+` | `xoxb-123-456-...` |
| Generic JWT | `eyJ[A-Za-z0-9-_]+\.[A-Za-z0-9-_]+` | `eyJhbGciOi...` |
| Private Key | `-----BEGIN.*PRIVATE KEY-----` | PEM key blocks |

Patterns are applied to:
- HTTPS request **bodies** (JSON, form-encoded, multipart)
- HTTPS request **headers** (`Authorization`, `X-Api-Key`, etc.)
- stdout / stderr **output streams**
- DNS query **subdomain labels**

### Adding Custom Patterns

You can extend the built-in pattern library in your `envtrap.json`:

```json
{
  "patterns": [
    {
      "name": "My Internal API Key",
      "regex": "myco_[a-zA-Z0-9]{32}",
      "severity": "block"
    }
  ]
}
```

## Shannon Entropy Analysis

Some secrets don't match known patterns — randomly generated tokens, symmetric encryption keys, or custom credentials. envtrap uses **Shannon entropy** to catch these.

Shannon entropy measures the randomness of a string on a scale of 0–8 bits per character:

- `"hello world"` → entropy ≈ 2.8 (low, normal text)
- `"abc123"` → entropy ≈ 2.6 (low)
- `"aB3!kR9mNq2Xp7v"` → entropy ≈ 4.1 (medium)
- `"sk_live_a4b4ecc7d4a4..."` → entropy ≈ 5.8 (**high** — flagged)

### Threshold Configuration

```json
{
  "entropy": {
    "enabled": true,
    "threshold": 5.0,
    "minLength": 16
  }
}
```

| Parameter | Default | Description |
|---|---|---|
| `threshold` | `5.0` | Minimum entropy score to flag (0–8) |
| `minLength` | `16` | Minimum string length before entropy check |
| `enabled` | `true` | Toggle entropy detection on/off |

## Secret Fingerprinting

When a secret is detected, envtrap never logs the raw secret value. Instead it computes a **SHA-256 fingerprint** of the value:

```
Fingerprint: SHA256:a4b4ecc7d4a47b3e2c1a9f8d...
```

This fingerprint:
- Can be used to track the same secret across multiple detection events
- Is safe to store in audit logs without exposing the actual credential
- Helps identify if the same key is being sent to multiple destinations

## Detection Flow

```
Intercepted Data
       │
       ▼
  ┌────────────────┐     Match ┌──────────────────┐
  │ Pattern Match  │──────────▶│  Policy Engine   │
  └────────┬───────┘           └──────────────────┘
           │ No match                  │
           ▼                    Block / Warn
  ┌────────────────┐
  │ Entropy Score  │
  │ > threshold?   │
  └────────┬───────┘
           │ Yes
           ▼
     Flagged & fingerprinted
```
