## Overview

envtrap runs your application inside a monitored child process context. It uses a combination of **local proxy routing**, **CommonJS monkey-patching**, and **ESM customization hooks** to intercept outbound leaks before they leave your process boundaries.

This approach is fundamentally different from static analysis:

| Approach | When | What It Sees |
|---|---|---|
| SAST | Build time | Source code patterns |
| envtrap | **Runtime** | **Actual data in flight, actual destinations** |

## How Interception Works

When you run `envtrap run node app.js`, envtrap orchestrates the security boundaries through the following steps:

```
                  ┌──────────────────────────────────────────┐
                  │          envtrap CLI (Spawner)           │
                  └────────────────────┬─────────────────────┘
                                       │
                Spawns with proxy env variables + import hooks
                                       │
                                       ▼
    ┌──────────────────────────────────────────────────────────────────────┐
    │ Node.js Application Process (Child)                                  │
    │                                                                      │
    │  ┌────────────────────┐          ┌────────────────────────────────┐  │
    │  │ Application Code   │─────────▶│ Patched Core Modules           │  │
    │  │ & npm Dependencies │          │ (dns, child_process)           │  │
    │  └────────────────────┘          └───────────────┬────────────────┘  │
    │                                                  │                   │
    │                                                  ▼                   │
    │  ┌────────────────────┐          ┌────────────────────────────────┐  │
    │  │ Streams & Outputs  │          │ Local MITM Proxy (127.0.0.1)   │  │
    │  │ (stdout / stderr)  │          │ (Decrypts TLS Outbound HTTPS)  │  │
    │  └─────────┬──────────┘          └───────────────┬────────────────┘  │
    │            │                                     │                   │
    └────────────┼─────────────────────────────────────┼───────────────────┘
                 │ Pipes                               │ Requests
                 ▼                                     ▼
      ┌──────────────────────────────────────────────────────────────────┐
      │                      envtrap Scanner Engine                      │
      │   • Entropy Screening   • Pattern Checks   • Path Exclusions     │
      └──────────────────────────────────────────────────────────────────┘
```

### 1. HTTP/HTTPS Decryption (MITM Proxy)
On startup, envtrap spawns an in-memory HTTP/HTTPS proxy server on `127.0.0.1` and points standard proxy environment variables (`HTTP_PROXY`, `HTTPS_PROXY`, `http_proxy`, `https_proxy`) to it. It also automatically sets `NODE_EXTRA_CA_CERTS` to point to our generated Root CA certificate.
All outbound network calls are routed through this proxy, decrypted, scanned for matching secrets, and either blocked (returning a `403` or `502` response) or forwarded to the upstream server.

### 2. CommonJS Monkey-patching
For applications or dependencies using CommonJS (`require()`), envtrap overrides `Module.prototype.require` inside the preload context.
When code loads `child_process` (or `node:child_process`) or `dns` (or `node:dns`), envtrap returns wrapped versions of these modules to intercept calls.

### 3. ESM Loader Customization Hooks
For modern ESM applications (`import`), envtrap registers Customization Hooks (`--import`) using Node's customization registry to hook module resolutions. It intercepts and redirects requests for `child_process` and `dns` to secure virtual module wrappers.

## Interception Channels

### Outbound Network (`network`)
* Decrypts HTTP/HTTPS requests in real-time.
* Scans headers, path URLs, and bodies for matches of active secrets.
* If a secret is matched and the mode is `"block"`, the request connection is terminated immediately.

### Subprocesses (`child_process`)
* Intercepts `exec`, `execSync`, `spawn`, `spawnSync`, `fork`, `execFile`, and `execFileSync`.
* Scans the environment variables (`options.env`) being passed to the child process.
* If active secrets are inherited by the child environment, envtrap blocks execution or throws a synchronous error in `"block"` mode.

### Name Resolution (`dns`)
* Hooks `lookup`, `resolve`, and all other host query promise endpoints.
* Scans target hostnames for active secrets.
* Scans subdomains using Shannon entropy to warn on possible DNS tunneling exfiltrations.

### Standard Outputs (`stdout` / `stderr`)
* Intercepts writes to standard output and error streams.
* Output containing active secrets is redacted to `[REDACTED: SHA256:<hash>]`.
* In `"block"` mode, detecting a leak terminates the process via `SIGTERM` immediately.

## Preload Script Setup

The hooks are automatically configured by envtrap spawner via:

```bash
NODE_OPTIONS="--import /path/to/envtrap/dist/hooks.mjs"
```

The hooks run **synchronously before any user code is executed**, guaranteeing that all core modules are safely wrapped before dependencies load.

---

## Path Exclusions Handling

If an operation originates from a path specified in `exclusions.paths` (checked using stack tracing):
* **Subprocess & DNS**: Suppresses detections/warnings.
* **Streams (stdout/stderr)**: Pre-redacts any output to `[REDACTED: PATH_EXCLUDED]` inside the hook before it is written to the pipe, preventing the outer spawner from raising alarms.
