## Overview

envtrap's pattern library and entropy detection are aggressive by design. In production environments, you will likely need to configure exclusions to prevent false positives from:

- Known-safe API domains (e.g., `api.stripe.com` — your intentional traffic)
- Non-sensitive high-entropy strings (e.g., UUIDs, nonces, hashes)
- Internal credential patterns that overlap with generic patterns

## Domain Allow-list

Domains in the allow-list skip body and header inspection for HTTPS requests:

```json
{
  "exclusions": {
    "domains": [
      "api.stripe.com",
      "api.openai.com",
      "sentry.io",
      "logs.datadoghq.com",
      "o123456.ingest.sentry.io"
    ]
  }
}
```

<Warning>
  Only add domains you explicitly control or fully trust. An attacker who registers `api.stripe.com.attacker.com` would still be blocked — envtrap matches the exact domain, not a substring.
</Warning>

## Path Exclusions

Exclude specific source file paths from triggering detections (useful for test suites or build scripts):

```json
{
  "exclusions": {
    "paths": [
      "test/**",
      "**/__tests__/**",
      "*.test.js",
      "*.spec.ts"
    ]
  }
}
```

<Note>
  Path exclusions use glob matching. envtrap resolves the call stack of each intercepted operation and checks if the call originated from a path matching the glob pattern.
</Note>

## Complete Exclusion Example

```json
{
  "exclusions": {
    "domains": [
      "api.stripe.com",
      "api.openai.com",
      "sentry.io"
    ],
    "paths": [
      "test/**",
      "**/__tests__/**"
    ]
  }
}
```
