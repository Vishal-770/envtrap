## What Are Channels?

A **channel** in envtrap represents a specific data-exfiltration path that can carry secrets out of your Node.js process. envtrap intercepts all five channels by default.

## Channel Configuration Format

Each channel can be configured with one of the following enforcement modes:

- `"block"`: If a leak is detected, the operation is prevented or blocked (and stdout/stderr kills the process).
- `"warn"`: If a leak is detected, it is logged (and stdout/stderr outputs are redacted), but the operation proceeds.
- `"off"`: Monitoring on this channel is disabled.

```json
{
  "channels": {
    "stdout": "warn",
    "stderr": "warn",
    "network": "block",
    "child_process": "warn",
    "dns": "block"
  }
}
```

---

## stdout Channel

Intercepts writes to standard output (`process.stdout`).

- **`"block"`**: If a loaded secret is detected, it terminates the process with exit code `1` immediately.
- **`"warn"`** (Default): If a loaded secret is detected, it redacts it to `[REDACTED: SHA256:<hash>]` on stdout and logs a warning.
- **`"off"`**: Completely disables interception on `stdout`.

---

## stderr Channel

Intercepts writes to standard error (`process.stderr`).

- **`"block"`**: If a loaded secret is detected, it terminates the process with exit code `1` immediately.
- **`"warn"`** (Default): If a loaded secret is detected, it redacts it to `[REDACTED: SHA256:<hash>]` on stderr and logs a warning.
- **`"off"`**: Completely disables interception on `stderr`.

---

## network Channel

Intercepts outbound TCP network traffic, running a local HTTP/HTTPS MITM proxy on `127.0.0.1`.

- **`"block"`** (Default): If a loaded secret is found in request headers, URL, or body, the connection is terminated and a `403 Forbidden` response is returned to the app.
- **`"warn"`**: If a secret is found, logs the leak, but forwards the request to the upstream target.
- **`"off"`**: Disables the proxy completely.

You can allow specific domains to bypass proxy checks using `exclusions.domains`.

---

## child_process Channel

Intercepts subprocess creation (`exec`, `spawn`, `fork`, etc.).

- **`"block"`**: Spawning is blocked and a synchronous error is thrown if the environment variables (`options.env`) passed to the child contain an active secret.
- **`"warn"`** (Default): Logs the subprocess creation leak warning and allows spawning.
- **`"off"`**: Disables environment checks on subprocesses.

---

## dns Channel

Intercepts hostname resolutions via `node:dns`.

- **`"block"`** (Default): Resolving a hostname that contains a loaded secret will fail the lookup immediately.
- **`"warn"`**: Logs the DNS leak but permits lookup resolution.
- **`"off"`**: Disables DNS monitoring.

*Note: High-entropy DNS subdomains are scanned as potential DNS tunneling exfiltrations and warn the user regardless of exact secret content.*
