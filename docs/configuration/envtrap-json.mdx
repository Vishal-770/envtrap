## Overview

Create an `envtrap.json` file in your project root to customize envtrap's behavior. All fields are optional — envtrap ships with sensible defaults that work out of the box.

```json
{
  "channels": {
    "stdout": "warn",
    "stderr": "warn",
    "network": "block",
    "child_process": "warn",
    "dns": "block"
  },
  "exclusions": {
    "domains": ["api.stripe.com"],
    "paths": ["test/**", "**/__tests__/**"]
  },
  "entropy": {
    "threshold": 3.5,
    "minLength": 12
  },
  "quiet": false,
  "logFile": "envtrap.log"
}
```

---

## Top-Level Options

### `channels`

**Type**: `object`

Defines the enforcement mode for each runtime interception channel. Each key must be set to one of `"block"`, `"warn"`, or `"off"`:

- `"block"` — Detected secrets block/terminate the action (and for stdout/stderr, kill the child process).
- `"warn"` — Detected secrets are logged, and stdout/stderr streams are redacted, but the execution continues.
- `"off"` — Monitoring on this channel is disabled completely.

| Channel | Default | Description |
|---|---|---|
| `stdout` | `"warn"` | Scans writes to standard output stream. Redacts secrets on match. |
| `stderr` | `"warn"` | Scans writes to standard error stream. Redacts secrets on match. |
| `network` | `"block"` | Runs the local MITM proxy to decrypt and scan outbound HTTP/HTTPS request bodies and headers. |
| `child_process` | `"warn"` | Intercepts spawn/exec commands and blocks/warns if active secrets are passed inside the subprocess environment variables (`options.env`). |
| `dns` | `"block"` | Intercepts `node:dns` queries to block lookups containing loaded secrets, or warn on high-entropy subdomains. |

```json
{
  "channels": {
    "network": "block",
    "stdout": "warn"
  }
}
```

---

### `exclusions`

**Type**: `object`

Exclude specific network destinations or file paths from checks to prevent false positives.

| Field | Type | Default | Description |
|---|---|---|---|
| `domains` | `string[]` | `[]` | Fully qualified domain names (FQDNs) to skip from proxy and network checks (e.g. `["api.stripe.com"]`). |
| `paths` | `string[]` | `[]` | Glob patterns of source files to exclude from stack checks. Intercepted events originating from these paths are ignored. |

```json
{
  "exclusions": {
    "domains": ["api.stripe.com"],
    "paths": ["test/**", "*.test.js"]
  }
}
```

---

### `entropy`

**Type**: `object`

Configure the Shannon entropy detector which screens environment variables on startup.

| Field | Type | Default | Description |
|---|---|---|---|
| `threshold` | `number` | `3.5` | Minimum Shannon entropy score (0 to 8) a string must have to be considered a secret. |
| `minLength` | `number` | `12` | Minimum character length for entropy scanning. |

```json
{
  "entropy": {
    "threshold": 3.5,
    "minLength": 12
  }
}
```

---

### `quiet`

**Type**: `boolean`  
**Default**: `false`

If set to `true`, suppresses the envtrap startup banner and real-time console alerts. Only prints the final summary of exfiltrations upon process exit.

```json
{ "quiet": true }
```

---

### `logFile`

**Type**: `string | null`  
**Default**: `null`

Optional path relative to the current working directory (or absolute path) to write structured JSONL events as leaks are detected.

```json
{ "logFile": "logs/envtrap-events.jsonl" }
```
