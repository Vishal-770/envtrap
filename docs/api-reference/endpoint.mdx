## `envtrap run`

The primary command. Wraps a Node.js process with envtrap's runtime interceptors.

```bash
envtrap run [options] <command> [args...]
```

### Arguments

| Argument | Description |
|---|---|
| `command` | The command to run (typically `node`) |
| `args` | Arguments to pass to the command |

### Options

| Flag | Default | Description |
|---|---|---|
| `-e, --env-file <path>` | `.env` | Path to a custom `.env` file to load secrets from |
| `-v, --verbose` | `false` | Enable verbose proxy and hook debug logging |
| `--no-mitm` | — | Disable HTTPS MITM proxy (faster startup, no network scan) |
| `--quiet` | `false` | Suppress startup banner and leak alerts (show summary only) |
| `--log-file <path>` | — | Path relative to CWD to write structured JSONL events |

### Examples

**Basic usage:**
```bash
envtrap run node app.js
```

**With arguments:**
```bash
envtrap run node --max-old-space-size=4096 server.js --port 3000
```

**Using custom environment file:**
```bash
envtrap run --env-file .env.production node app.js
```

**Disable MITM proxy network scanning:**
```bash
envtrap run --no-mitm node app.js
```

**Running with quiet and logging output to a file:**
```bash
envtrap run --quiet --log-file logs/envtrap-run.jsonl node app.js
```

## `envtrap check`

Validates your `envtrap.json` configuration file in the current working directory.

```bash
envtrap check
```

**Output on success:**
```text
✅ Configuration is valid.
```

**Output on error:**
```text
Configuration Validation Failed:
  - [$.channels.network] Invalid mode "invalid-mode". Valid modes: block, warn, off
```

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | Process exited normally (no detections in block mode, or validations passed) |
| `1` | A secret was detected and blocked (mode=block), or validation failed |
| Forwarded | The wrapped process's own exit code (when no block/leak triggers exit) |
